# 專案概述
Hexagon Vocabulary & Grammar，單頁應用程式 (SPA)，結合六角形地圖介面、無冷卻間隔重複系統 (SRS) 與 Firebase 雲端同步。

# 檔案導引
開發前請務必讀取 ROADMAP.md 了解當前任務進度與下一個目標。

# 技術棧
- 前端：純 HTML, CSS (自訂屬性), Vanilla JavaScript。無使用前端框架。
- 後端：Firebase (Auth, Firestore) 版本 10.12.0 (compat 模式)。
- 圖形：HTML5 Canvas API。
- 語音：Web Speech API (SpeechSynthesis, SpeechRecognition)。

# 核心業務邏輯
1. 狀態與存檔管理
   - 支援多存檔 (`appSaves`)，各自獨立保存 tokens、grid、storageData、checkInHistory、unlockedRules 以及 自選語音設定。
   - 雙重儲存機制：登入狀態同步至 Firestore，未登入存於 LocalStorage。
2. 測驗系統 (SRS)
   - 抽取規則：L5 隨機取 5 個；L1-L4 取進入該 Level 時間 (`levelUpdatedAt`) 最舊的前 5 個；L0 取題庫新字。
   - 防禦規則：絕對禁止加入任何 SRS 天數冷卻或延遲邏輯，單純依據時間戳記排序。
   - 升降級規則：過關升級並更新時間戳記；L1-L4 失敗等級不變且不更新時間戳記；L5 失敗降為 L4 並更新時間戳記；L0 失敗不做任何移位操作，保留在原順序以便下次優先測驗。
3. 視覺與地圖
   - 地圖格子背景色對應單字詞性 (POS)。
   - 等高線 (Level 2-5) 需判斷相鄰且同級/高級的六角形，將外框融合繪製。

# 任務執行規範 (Workflow)
每次執行開發任務時，請嚴格遵守以下流程：
1. 確立目標：接收指令後，先釐清並更新 ROADMAP.md 中的當前目標。
2. 讀取：檢視相關模組檔案與依賴關係。
3. 實作：修改程式碼。
4. 驗證：修改完成後，主動檢查邏輯正確性與跨裝置相容性。
5. 標記與押期：確認無誤後，將 ROADMAP.md 中的任務標記為 [x] 並押上完成日期 (如 `[x] (2026-06-10)`)。嚴禁修改、竄改或刪除已完成的歷史任務。
6. 自動歸檔：若「當前任務」區塊中的已完成項目累積超過 5 條，請主動將其剪下並移至最下方的 `## 歷史紀錄 (Archive)` 區塊，維持版面整潔。
7. 回報：向使用者回報任務完成並簡述修改重點。

# 開發與溝通規範
- 直接說明重點，文字必須簡潔精準，避免堆砌詞藻。
- 絕對禁止使用任何表情符號。
- 避免使用對比式論述 (如：不是...而是...)。
- 程式碼需符合現有 Vanilla JS 結構，避免引入不必要的第三方庫。
- 推進進度時，主動考慮檔案模組化與架構的整潔性。
